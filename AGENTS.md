## Дозволи на виконання

### ✅ Виконую самостійно (без питань)

```bash
# Git — тільки мої файли
git status / git add / git commit / git push
# в ~/.openclaw/workspace/ ТІЛЬКИ

# Моніторинг і читання
ps aux
tail -f / tail -20 (будь-які логи)
journalctl -f --no-pager
systemctl status * (будь-який сервіс — тільки читання)
free -m / df -h / uptime / top

# Файли
read/write все в ~/.openclaw/workspace/
читання логів (не видалення)
tree / find / cat / grep / ls
```

### ⚠️ Питаю перед виконанням

```bash
# Продакшн сервіси — це живий бот клієнтів!
systemctl restart insilver-v3
systemctl stop insilver-v3
systemctl start insilver-v3

# Git в продакшн репо
git push в ~/.openclaw/workspace/insilver-v3/
git commit в insilver-v3/ (будь-які зміни)

# Системні зміни
sudo * (окрім systemctl status)
редагування будь-яких .env файлів
rm / rmdir (будь-яке видалення)
pip install / apt install
зміни конфігурацій сервісів
```

### ❌ Заборонено назавжди

```bash
reboot / shutdown / halt / poweroff
rm -rf з широкими масками
chmod 777 на будь-яких файлах
зміна systemd unit файлів

```

## InSilver Docs — Обов'язковий принцип

**При кожній новій фічі/зміні:**
1. Розробляю → одразу думаю "як пояснити Владу?"
2. Тестую → перевіряю зрозумілість документації  
3. Оновлюю InSilver Docs під нову функцію
4. Даю Сашку на review → передача Владу

**Мета:** Влад завжди знає як користуватися всіма можливостями без звернень за допомогою.

**Документація = частина продукту, не опціональне додавання.**

## Перемикання моделей

Ти вмієш перемикати власну активну модель прямо під час розмови через `config.patch`.

### Доступні моделі:
- **haiku** → `claude-haiku-4-5-20251001` (швидка, дешева, звичайні задачі)
- **sonnet** → `claude-sonnet-4-6` (збалансована, складні задачі)
- **opus** → `claude-opus-4-6` (найпотужніша, повільна/дорога)

### Коли користувач каже "переключись на haiku/sonnet/opus":
→ Одразу виконай (без підтвердження):
```bash
openclaw config.patch '{"ai": {"model": "<МОДЕЛЬ>"}}'
```
Відповідай: `✓ Переключено на <назва>. Активна з наступного повідомлення.`

## Команда "чкп" (розумний чекпоінт)

Коли Сашко пише **"чкп"** — виконай ці кроки по порядку, без зупинок і без зайвих питань:

### Крок 1 — Розумний memory flush
Зберегти в `memory/YYYY-MM-DD.md` **технічний контекст**:
```
## Technical Context (швидке відновлення)
- Working state: [останній робочий стан]
- Recent fixes: [що щойно виправили]
- Issues encountered: [з якими проблемами зустрічались]
- Next priorities: [конкретні наступні кроки]
- Key files changed: [шляхи до змінених файлів]

## Architecture insights 
- [важливі знахідки про структуру коду]

## Cost optimizations
- [застосовані оптимізації і економія]
```

### Крок 2 — Оновити документацію
Онови ці файли:
- `~/.openclaw/workspace/insilver-v3/DEV_CHECKPOINT.md` — стан проекту
- `~/.openclaw/workspace/BOOTSTRAP.md` — якщо змінилась структура

### Крок 3 — Git commit і push  
```bash
cd ~/.openclaw/workspace/insilver-v3 && git add -A && git commit -m "[авто-опис]" && git push
cd ~/.openclaw/workspace && git add -A && git commit -m "docs: checkpoint [дата]" && git push
```

### Крок 4 — Звіт + розумне скидання
Надішли підсумок і **НЕ робити /new автоматично**:
```
🐱 ЧКП [дата] - РОЗУМНИЙ РЕЖИМ

✅ Зроблено: [список]  
📁 Git: [commits]

🚀 Наступні кроки: [priorities]
```

4. Тільки після цього — звітуй
