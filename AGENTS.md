## Перемикання моделей

Ти вмієш перемикати власну активну модель прямо під час розмови через `config.patch`.

### Доступні моделі:
- **haiku** → `claude-haiku-4-5-20251001` (швидка, дешева, звичайні задачі)
- **sonnet** → `claude-sonnet-4-6` (збалансована, складні задачі)
- **opus** → `claude-opus-4-6` (найпотужніша, повільна/дорога)

### Коли користувач каже "переключись на haiku/sonnet/opus":
→ Одразу виконай (без підтвердження):
```bash
openclaw config.patch '{"ai": {"model": "<МОДЕЛЬ>"}}'
```
Відповідай: `✓ Переключено на <назва>. Активна з наступного повідомлення.`

## Команда "чкп" (розумний чекпоінт)

Коли Сашко пише **"чкп"** — виконай ці кроки по порядку, без зупинок і без зайвих питань:

### Крок 1 — Розумний memory flush
Зберегти в `memory/YYYY-MM-DD.md` **технічний контекст**:
```
## Technical Context (швидке відновлення)
- Working state: [останній робочий стан]
- Recent fixes: [що щойно виправили]
- Issues encountered: [з якими проблемами зустрічались]
- Next priorities: [конкретні наступні кроки]
- Key files changed: [шляхи до змінених файлів]

## Architecture insights 
- [важливі знахідки про структуру коду]

## Cost optimizations
- [застосовані оптимізації і економія]
```

### Крок 2 — Оновити документацію
Онови ці файли:
- `~/.openclaw/workspace/insilver-v3/DEV_CHECKPOINT.md` — стан проекту
- `~/.openclaw/workspace/BOOTSTRAP.md` — якщо змінилась структура

### Крок 3 — Git commit і push  
```bash
cd ~/.openclaw/workspace/insilver-v3 && git add -A && git commit -m "[авто-опис]" && git push
cd ~/.openclaw/workspace && git add -A && git commit -m "docs: checkpoint [дата]" && git push
```

### Крок 4 — Звіт + розумне скидання
Надішли підсумок і **НЕ робити /new автоматично**:
```
🐱 ЧКП [дата] - РОЗУМНИЙ РЕЖИМ

✅ Зроблено: [список]  
📁 Git: [commits]
🧠 Технічний контекст збережено в memory/
💰 Сесія продовжується (економія ~$0.70)

🚀 Наступні кроки: [priorities]
```

## Команда "фікс" (швидке збереження успіху)

Коли Сашко пише **"фікс"** — зберегти успішні налаштування/зміни:

**Дії:**
1. Створити backup важливих конфігів (якщо треба)
2. Додати документацію що було виправлено  
3. Git commit з описовим повідомленням
4. **Мінімум токенів** - тільки необхідне

**Приклад сьогодні:**
- `TELEGRAM_SUCCESS.md` - звіт
- `openclaw.json.telegram-working` - backup  
- Commit: "🎯 SUCCESS: OpenClaw Telegram integration working"

**Коли використовувати:**
- ✅ Щось налаштували і працює
- ✅ Виправили складну проблему
- ✅ Треба зафіксувати перед наступним кроком  
- ✅ Документувати для майбутнього

## Автостарт сесії

При **кожному** першому повідомленні після `/new` або перезапуску:
1. Прочитай `~/.openclaw/workspace/insilver-v3/DEV_CHECKPOINT.md`
2. Виконай `sudo systemctl status insilver-v3 | head -5`
3. Привітайся:
```
🐱 Кіт з DEV_CHECKPOINT [дата]
Останнє: [що зроблено]
Бот: [active/inactive]
Далі: [наступні задачі з чекпоінту]
```

## Правило: перед restart — перевір дублікати

Перед **будь-яким** `systemctl restart insilver-v3` або запуском `main.py`:
```bash
ps aux | grep main.py | grep -v grep
```
Якщо є більше одного процесу — спочатку kill всі, потім запускай. Ніколи не запускай поверх живого процесу.

---

## Правило: "готово" тільки після тесту

Ніколи не кажи Сашку "готово" або "виправлено" без самостійної перевірки.

Після кожної зміни:
1. Перезапусти бота
2. Перевір логи — `tail -20 ~/.openclaw/workspace/insilver-v3/logs/bot.log`
3. Протестуй саме ту функцію яку міняв
4. Тільки після цього — звітуй

Якщо не можеш протестувати сам — **скажи прямо** що саме треба перевірити Сашку і як.

## Правило: мінімізуй tool calls

**Оптимізуй кількість викликів** — менше кроків = швидша відповідь = менші витрати.

### Батчування команд:
```bash
# ❌ Замість множинних exec:
exec("cmd1")  
exec("cmd2")
exec("cmd3")

# ✅ Один батч:
exec("cmd1 && echo '===' && cmd2 && echo '===' && cmd3")
```

### Комплексний аналіз:
```bash  
# ❌ Замість окремих ls, grep, awk
# ✅ Один pipe з усіма операціями
exec("ls -la path/ | tee /tmp/data && grep pattern /tmp/data | awk commands")
```

**Мета:** Звичайний запит ≤3-4 tool calls, складний ≤6 tool calls.
